# core_directory.md 精简模板

用于生成面向笔记记录的 `core_directory.md`。`core_directory.md` 是对 `directory_graph` 的文字补充：只描述顶层目录的职责，不展开 sub folder，不列具体文件，不加入完整报告中的入口点、关键文件列表、文件连接、阅读顺序等内容。

```markdown
以下结构由 AI 生成。

## 核心目录结构

- 📁 **<top-level-directory>/** — <用一段自然中文总结该顶层目录的完整功能范围。描述应比普通一句话目录说明更充分，可以覆盖该目录中的主要子系统、典型职责、与其它模块的关系，但不要列出 sub folder。>

- 📁 **<another-top-level-directory>/** — <同上，只描述顶层目录。>

## 目录依赖图

说明：

- **节点**：一个顶层 package 目录。
- **箭头方向**：`A → B` 表示 `A` 目录下的文件在静态代码中依赖 `B` 目录下的文件。
- **箭头数字**：从源目录到目标目录的文件级依赖边数量。数字越大，说明两个目录之间的静态耦合越强。
- **橙色节点**：更偏上层入口或编排层，向外依赖较多。
- **蓝色节点**：更偏底层基础设施，被其他目录依赖较多。
- **紫色节点**：输入和输出依赖较平衡，通常起连接作用。
```

## 写作要求

- `core_directory.md` 默认采用该精简模板，而不是长篇逐文件摘要。
- `core_directory.md` 是对 `directory_graph` 的补充，重点解释每个顶层目录“负责什么、包含哪些能力、在项目里扮演什么角色”。
- 只列顶层目录，不列 sub folder，不写具体文件条目；除非用户明确要求展开子目录。
- 每个顶层目录使用一段稍长的自然中文说明，通常 1-2 句话，信息量应高于普通目录树的一句话简介。
- 说明文字使用自然中文；路径、目录名、模型名和必要术语保持原样。不要用斜体包裹目录说明，普通正文更适合做笔记。
- `<img>` 的 `src` 优先使用用户笔记可引用的相对路径；如果用户没有指定笔记图片目录，则使用生成出来的 PNG 路径，例如 `mermaid_elk_render/directory_graph_elk.png`。
- 如果没有成功渲染 PNG，仍保留“目录依赖图”小节，但用一句中文说明图片未生成，并指向 `directory_graph.mmd`。
